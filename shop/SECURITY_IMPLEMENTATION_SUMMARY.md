  - JavaScript protocol attacks

- **Protected:** All POST, PUT, PATCH, DELETE requests
- **Frontend Integration:** Include `X-CSRF-Token` header

### 7. ✅ Audit Logging

## 🚀 How to Test

### 1. Test Email Verification (Development Mode)

```bash
# 1. Register a new user
POST http://localhost:8080/api/auth/register
{
  "name": "Test User",
  "email": "test@example.com",
  "password": "password123"
}

# 2. Check server logs - you'll see:
#    "Verification email sent to: test@example.com"
#    "Action URL: http://localhost:5173/verify-email?token=xxx"

# 3. Copy the verification URL from logs and open in browser
# 4. Email will be verified automatically

# 5. Try resending verification email:
POST http://localhost:8080/api/auth/resend-verification
Headers: Authorization: Bearer <your-token>
```

### 2. Test Account Lockout

```bash
# Try logging in with wrong password 5 times:

curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"wrongpassword"}'

# Attempt 1: "Invalid email or password. 4 attempts remaining."
# Attempt 2: "Invalid email or password. 3 attempts remaining."
# Attempt 3: "Invalid email or password. 2 attempts remaining."
# Attempt 4: "Invalid email or password. 1 attempts remaining."
# Attempt 5: "Account locked due to multiple failed login attempts..."

# Check server logs for account locked email (simulated in dev mode)

# Wait 15 minutes OR reset password to unlock
```

### 3. Test CSRF Protection

```bash
# 1. Get CSRF token
curl http://localhost:8080/api/csrf-token

# Response: { "success": true, "data": { "csrfToken": "xxx" } }

# 2. Make protected request WITHOUT token (SHOULD FAIL)
curl -X POST http://localhost:8080/api/cart/add \
  -H "Content-Type: application/json" \
  -d '{"productId":"123"}'

# Response: 403 - "Invalid CSRF token"

# 3. Make protected request WITH token (SHOULD SUCCEED)
curl -X POST http://localhost:8080/api/cart/add \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: <token-from-step-1>" \
  -d '{"productId":"123"}'
```

### 4. Test XSS Sanitization

```bash
# Try injecting malicious script:
POST http://localhost:8080/api/products
{
  "title": "<script>alert('XSS')</script>Laptop",
  "description": "<img src=x onerror=alert('XSS')>Great laptop"
}

# Check response - scripts should be removed:
{
  "title": "Laptop",
  "description": "Great laptop"
}
```
